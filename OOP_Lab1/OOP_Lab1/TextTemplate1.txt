1.	Что такое .Net Framework и из чего он состоит?
Microsoft.Net Framework — это платформа,
которая нужна для функционирования многих программ написанных по технологии «.Net».
В его состав входит библиотека классов (FCL) и среда исполнения (CLR)

2.	Что такое CLR, FCL/BCL, CLI, IL? 
CLR (Common Language Runtime) — это исполняющая среда для байт-кода CIL , в который компилируются программы, написанные на.NET-совместимых ЯП.

В состав платформы.NET входит библиотека классов Framework Class Library (FCL) – элементом этой библиотеки является базовая библиотека классов Base Class Library (BCL).
В BCL – входят классы для работы со строками, коллекциями данных, поддержки многопоточности.

CLI - интеxрфейс командной строки

IL - промежуточный язык.

3.	Пояснить работу JIT-компилятора?
Виртуальная машина вызывает скомпилированный код метода напрямую, не интерпретируя его.
Следовательно, не требует большого использования памяти и процессорного времени.
Это в основном ускоряет производительность приложения.

4.	Что такое CTS (Common Type System)?
Common Type System (CTS) — это фундаментальный компонент платформы .NET, который определяет, как типы объявляются, используются и управляются в приложениях .NET.
CTS можно рассматривать как набор правил и рекомендаций, которые обеспечивают согласованность и взаимодействие между различными языками программирования в экосистеме .NET.

5.	Какие аспекты поведения определяет тип System.Object?
Тип System.Object в C# является базовым классом для всех типов и классов в .NET.
Он определяет несколько ключевых методов, которые могут быть переопределены в производных классах для изменения их поведения.

6.	Что находится в mscorlib dll?
mscorlib.dll — это основная библиотека классов для .NET Framework, содержащая фундаментальные типы, такие как System.Object,
System.String, System.Int32, коллекции, потоки, базовые структуры данных и другие базовые классы.

7.	Что такое «сборка»? Из чего состоит сборка .NET?
Сборка — это единица развертывания и компоновки в .NET. Она состоит из:
	Манифеста (содержит метаданные о сборке)
	MSIL-кода (среднеуровневый код, который JIT-компилятор превращает в машинный код)
	Метаданных (описание типов, методов, зависимостей)
	Ресурсов (неуправляемые данные, такие как изображения, файлы и т.д.)

8.	 Какие виды сборок существуют? 
Приватные сборки: Используются только в одном приложении.
Общие сборки: Сборки, хранящиеся в GAC и доступные для всех приложений.

9.	Что такое assembly manifest?
Манифест сборки — это метаданные, которые описывают содержимое сборки, её версии, зависимости и информацию о безопасности.

10.	Что такое GAC?
GAC (Global Assembly Cache) — это глобальный кэш, где хранятся общие сборки, доступные для нескольких приложений на компьютере.
	
11.	Чем managed code отличается от unmanaged code
Managed code — код, выполняемый под управлением среды CLR, которая занимается управлением памятью, безопасностью и обработкой исключений.
Unmanaged code — код, который выполняется непосредственно процессором и не управляется CLR (например, код на C++).

12.	Как и для чего определен метод Main?
Метод Main является точкой входа для выполнения программы.
Он может быть определен с параметрами или без, а также может возвращать значение int или void.

13.	Варианты использования директивы using в C#.
Используется для подключения пространств имен.
Для реализации интерфейса IDisposable.

14.	Как связаны между собой сборки и пространства имен?
Пространства имен группируют типы, а сборки — это физические контейнеры для типов. Одно пространство имен может содержать типы, распределенные по нескольким сборкам.

15.	Что такое примитивные типы данных? Перечислите их.
Примитивные типы — это встроенные типы данных в C#. К ним относятся: byte, sbyte, short, ushort, int, uint, long, ulong, float, double, char, bool, decimal.

16.	Что такое ссылочные типы? Какие типы относятся к ним?
Ссылочные типы хранят ссылку на объект в памяти. К ним относятся: string, классы, массивы, интерфейсы, делегаты.

17.	Какие типы относятся к типам-значениям? 
Типы-значения хранят данные непосредственно. К ним относятся: все примитивные типы (например, int, bool, char), структуры, перечисления (enum)

18.	 В чем отличие между ссылочными и значимыми типами данных?
Ссылочные типы хранят ссылку на объект, а значимые типы хранят сами данные. Значимые типы располагаются в стеке, а ссылочные — в управляемой куче.

19.	Что такое упаковка и распаковка значимых типов?
Упаковка (boxing) — это процесс преобразования значимого типа в ссылочный, например, int в object.
Распаковка (unboxing) — обратный процесс, извлечение значения из ссылочного типа в значимый.

20.	В чем заключается разница между int и System.Int32? double и System.Double и т.д.?
1-ый вариант является базовым для ЯП C#, 2-ой для платформы .NET.

21.	Для чего используется тип dynamic?
Тип dynamic позволяет обойти статическую проверку типов во время компиляции и проверять тип во время выполнения.

22.	В чем заключается главное отличие между var и dynamic?
var — это неявно типизированная переменная, и её тип определяется на этапе компиляции.
dynamic — это динамический тип, и его проверка происходит во время выполнения.

23.	Что такое неявно типизированная переменная?

24.	Для чего используют Nullable тип?
Nullable-типы позволяют переменной значимого типа принимать значение null.

25.	Как объявить строковый литерал? Какие операции можно выполнять со строкой?
Cтроковые литералы объявляются с использованием двойных кавычек.
Операции: конкатенация, сравнение, поиск, замена, форматирование, разбиение и т.д.

26.	Какие есть способы для задания и инициализации строк?
Строковый литерал, конструктор.

27.	Какие методы есть у типа String?
Методы типа String: Length, Substring(), Replace(), IndexOf(), ToLower(), ToUpper(), Trim(), Contains(), Split(), Join() и другие.

28.	В чем отличие пустой и null строки?
Пустая строка ("") — это строка длиной 0. Null строка — это отсутствие строки (null).

29.	Как можно выполнить сравнение строк?
Строки можно сравнивать с помощью методов Equals(), оператора == или метода Compare().

30.	В чем отличие типов String и StringBuilder?
String — это неизменяемый тип, при изменении строки создается новая строка.
StringBuilder — изменяемый тип, предназначен для частых операций изменения строки.

37.	Что такое локальная функция?  Какова область ее видимости?
Локальная функция — это функция, определенная внутри другой функции. Она доступна только в пределах функции, в которой определена.

38.	 В чем разница между кодом, заключенным в блок checked и кодом, заключенным в блок unchecked? 
checked проверяет переполнение при арифметических операциях.
unchecked игнорирует переполнение.

39.	Какой контекст (checked/unchecked) применяется по умолчанию? Как можно переопределить это поведение? 
По умолчанию контекст unchecked. Его можно переопределить, используя блоки checked и unchecked или ключевые слова в выражениях.

40.	Для чего используется ключевое слово fixed? Каковы особенности его использования?
Ключевое слово fixed используется для фиксации указателя на объект, чтобы предотвратить перемещение объекта в памяти, что необходимо при работе с указателями в управляемом коде C#.
